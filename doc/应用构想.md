# 应用构想

基于当前技术栈（Tauri 2 + Vue 3 + Node 侧车 + LangChain）的若干应用方向，用于拓展场景与积累可展示案例。每个构想都充分利用「桌面壳 + Node 能力 + 前端交互 + LLM/链」的组合优势。

---

## 1. 本地 RAG 知识库 / 文档助手

**思路**：用户选择本地文件夹或拖入文档，侧车用 LangChain 做分块、embedding（可接本地模型或 API），向量存 SQLite（或 Chroma/本地文件）；前端检索、问答、引用溯源。

**实际场景举例**：
- 把**产品手册、SOP、合同模板**丢进一个库，问：「报销流程是什么」「合同里违约金条款怎么写的」→ 返回答案并高亮出处段落、可点击跳转到原文件。
- **技术/API 文档**：导入项目文档或 Confluence 导出，问「xxx 接口的鉴权方式」「错误码 5003 怎么处理」→ 回答带文档链接或章节引用。
- **个人笔记/博客草稿**：本地 Markdown 文件夹建库，用自然语言检索「上周写的关于缓存的那段」「和 A 项目相关的待办」→ 相当于「本地版 Ctrl+K 知识库」。

**技术亮点**：Tauri 本地文件与托盘；侧车 LangChain RAG + embedding + better-sqlite3；Vue 知识库管理、对话与引用高亮。

**可展示点**：RAG 全链路、本地数据与隐私、多进程架构、桌面端产品化。

**潜力**：★★★★☆

## 2. 智能自动化工作流 / 内部小 Agent

**思路**：用自然语言描述任务，LangChain Agent 规划步骤、调用工具；侧车提供读文件、写文件、执行脚本、查数据库等能力，Vue 做流程编排与执行日志。

**实际场景举例**（以下可按需选做、后续再完善）：

- **周报/日报汇总**：「把这周 Jira 完成项 + Git 提交记录按模板生成周报」→ Agent 调工具取数据、填模板，输出 Markdown 或写进 Notion/飞书。
- **批量重命名/整理**：「把这个目录下所有图片按拍摄日期重命名并移到按年月分的子文件夹」→ 读目录、解析 exif、写文件。
- **模板填充 / 文档生成**：「按这份简历模板，根据我下面几段经历生成一版正文」或「按周报模板，把这几条要点扩写成一段」→ 读模板 + 用户输入，输出可复制或导出为 Markdown/Word。
- **项目依赖检查**：选项目目录，「有没有已知漏洞的依赖、能升级的 major 版本」→ 读 package.json / requirements.txt，调 npm audit / npm outdated，Agent 汇总成可读报告。

*开发向（侧车可提供的工具/能力）*：
- **Commit message 生成**：选若干已 stage 的文件或 diff，「根据改动生成一条 conventional commit」→ 读 diff，LLM 生成 message，可写入 .git/ 或仅复制。
- **单测/用例建议**：指定一个源文件或函数名，「给这个函数列几条单元测试用例（输入/预期）」→ 读源码，输出用例列表或 Jest/vitest 风格骨架。
- **Mock 数据 / 接口示例**：给出接口签名或 JSON Schema，「生成 5 条 mock 数据」→ 输出 JSON/TS，可写入文件或复制。
- **日志/报错归类**：粘贴一段日志或报错栈，「这段是啥问题、可能原因、排查步骤」→ 纯 LLM 分析，可选「把同类错误记到本地清单」方便后续检索。
- **重复/大文件扫描**：选目录，「找出重复文件（按 hash）或超过 50MB 的文件，列个清单」→ 侧车遍历、算 hash/大小，输出清单或导出 CSV。
- **环境/配置检查**：选项目根目录，「检查 .env.example 和代码里用的 env 是否一致、有没有漏写的」→ 读 .env* 与代码中的 process.env / import.meta.env，对比并列出差异。

*用户向（终端用户可直接用的例子）*：
- **剪贴板/摘录整理**：用户粘贴多段文字，「按主题分个类并生成一份摘要/待办」→ 输出结构化笔记或 Markdown。
- **书签/链接归档**：导入浏览器书签或一列 URL，「按主题分类并生成一个 Markdown 阅读清单」→ 分类 + 简短描述，导出 MD。
- **多文件关键词统计**：选文件夹，「统计这些文件里出现最多的 20 个词（可排除停用词）」→ 读文本、分词、统计，输出词云数据或列表。

**技术亮点**：Node 侧车 LangChain Agent + 自定义 Tools；Tauri 安全边界与可选全局快捷键；Vue 工作流列表、步骤可视化、历史与重跑。

**可展示点**：Agent 与 Tool Use、多工具编排、桌面与系统集成。

**潜力**：★★★★★

## 3. 本地多模型对话 / 模型切换台

**思路**：侧车对接 Ollama、本地 OpenAI 兼容 API；LangChain 统一抽象与可选 chain；前端多会话、模型切换、流式输出与历史持久化。（*若当前不做「对话产品」，可暂缓。*）

**实际场景举例**：本地用 Ollama 跑 Llama/Qwen，或接自建/内网 API；多会话、选模型、流式打字机效果；会话与配置存 Tauri Store。

**技术亮点**：多后端统一抽象、流式、桌面端状态与 Store/DB 整合。

**可展示点**：LLM 抽象、流式体验、桌面端配置与持久化。

**潜力**：★★★☆☆

## 4. 代码与仓库分析 / 文档生成器

**思路**：用户选择本地仓库或目录，侧车用 LangChain 做代码理解、生成 README/注释/变更说明；前端展示树形结构、diff 预览与导出。

**实际场景举例**：
- **接手老项目**：选仓库根目录，问「这个项目是干什么的、主入口在哪、依赖哪些服务」→ 基于代码与目录结构生成一份 Onboarding 文档。
- **补文档**：选一个模块，一键「为这个模块生成 README：功能、入参出参、使用示例」→ 读源码 + 生成 Markdown，可编辑后写回或导出。
- **变更说明**：对比两个分支或两个目录，生成「本次改动涉及哪些模块、建议的 CHANGELOG 条目」→ 给 Code 模型 diff 或文件列表，输出结构化说明。

**技术亮点**：Tauri 目录选择与文件树；侧车 LangChain 代码理解与生成、可选 Git；Vue 树形、预览、导出。

**可展示点**：代码级 NLP、工程化流水线、本地/隐私/离线工具。

**潜力**：★★★★☆

## 5. 会议/笔记助手（转写 + 总结 + 待办）

**思路**：侧车集成语音转写（本地 Whisper 或云端 API），转写结果经 LangChain 做总结、待办提取、关键词；Tauri 常驻托盘、快捷录入、与系统日历/提醒的集成入口。

**实际场景举例**：
- **会议纪要**：录完会议 → 自动转写 → 生成「会议摘要、决议、待办、责任人」→ 导出 Markdown 或插入笔记软件。
- **灵感速记**：托盘一键开始录音，说完停 → 转写 + 自动打标签（如「需求」「Bug」「想法」）+ 存到本地笔记库，支持后续用 RAG 检索。
- **访谈/播客整理**：长音频分段转写 + 每段小结 + 全文关键词与时间戳，方便剪片或写稿。

**技术亮点**：侧车转写 pipeline + LangChain 摘要与信息抽取；Tauri 常驻、托盘、通知、快捷键；Vue 录音控制、转写与待办展示。

**可展示点**：多模态输入、NLP 信息抽取、常驻型桌面应用。

**潜力**：★★★★☆

---

## 6. 内部数据查询与报表小助手

**思路**：连接本地/内网数据库，用户用自然语言提问；侧车 LangChain Text-to-SQL，执行后返回表格或图表配置，前端 ECharts/表格渲染。（*偏 BI/内部工具，若当前用不到可暂缓。*）

**实际场景举例**：连 SQLite/PostgreSQL，问「上月销售额前 10 的客户」「各部门人数分布」→ 生成 SQL、执行、表格/饼图展示；连接信息存 Store。

**技术亮点**：NL2SQL、连接池与安全、桌面端无跨域。

**可展示点**：NL2SQL、数据安全与本地部署。

**潜力**：★★★☆☆

### 实现与差异化（原表微调）

| 构想         | 实现难度 | 对栈的覆盖 | 差异化程度 |
|--------------|----------|------------|------------|
| 本地 RAG     | 中       | 高         | 高         |
| Agent 工作流 | 中高     | 高         | 高         |
| 多模型对话   | 中       | 高         | 中         |
| 代码/文档    | 中       | 中高       | 高         |
| 会议/笔记    | 中高     | 高         | 高         |
| 数据查询     | 中       | 中高       | 中高       |

### 潜力 × 有趣 × 学习价值（重新评估）

| 构想         | 潜力 | 有趣程度 | 学习到更多知识 |
|--------------|----------|----------|----------------|
| 本地 RAG     | ★★★★☆   | ★★★★☆   | ★★★★★（RAG/向量/检索全链路） |
| Agent 工作流 | ★★★★★   | ★★★★★   | ★★★★★（Agent/规划/工具编排） |
| 多模型对话   | ★★★☆☆   | ★★★☆☆   | ★★★☆☆（LLM 抽象与流式） |
| 代码/文档    | ★★★★☆   | ★★★★☆   | ★★★★☆（代码理解与生成） |
| 会议/笔记    | ★★★★☆   | ★★★★☆   | ★★★★☆（多模态、信息抽取） |
| 数据查询     | ★★★☆☆   | ★★★☆☆   | ★★★☆☆（NL2SQL、BI 向） |

**有趣程度**：做出来是否有成就感、是否愿意长期维护。  
**学习价值**：能否系统性学到 RAG、Agent、多模态、代码理解等可迁移知识。

优先建议：**Agent 工作流** 与 **本地 RAG** 二选一或都做；**代码/文档**、**会议/笔记** 按兴趣与时间选一个。多模型对话、数据查询可留作后续扩展。
